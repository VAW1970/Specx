O que é o Specx?

O Specx é uma aplicação web para desenvolvimento assistido por IA seguindo a metodologia SDD (Specification-Driven Development), utilizando modelos de IA rodando localmente via Ollama.

Em vez de dar comandos vagos para a IA, o Specx oferece uma estrutura organizada onde você define:

O QUE fazer
SPEC.md — Especificação do projeto
COMO fazer
PLAN.md — Plano de implementação
O QUE fazer AGORA
TASK.md — Lista de tarefas

Primeiros Passos

Pré-requisito: O Ollama deve estar instalado e rodando (ollama serve).
1. Iniciar o aplicativo

No Windows, basta clicar duas vezes no arquivo start.bat. No terminal:

cd specx
pip install -r requirements.txt
python app.py
2. Abrir no navegador

Acesse http://localhost:5000

3. Criar seu primeiro workspace

Clique em "Novo Workspace" na página inicial e dê um nome ao seu projeto.

Workspaces

Um workspace é um container para todo o contexto do seu projeto. Ao criar um workspace, o Specx gera automaticamente:

  • workspace.json — Metadados do projeto
  • sdd/ — Documentos SPEC.md, PLAN.md, TASK.md
  • rules/ — Arquivo de regras padrão
  • memory/ — Diretórios para bancos de memória
  • gears/ — Perfis GEARS (criados pelo usuário)
Dica: Cada workspace é independente. Você pode ter vários projetos ao mesmo tempo.

Documentos SDD

O coração do Specx são os três documentos SDD. Cada um tem um papel específico:

Arquivo Ícone O que define Exemplo
SPEC.md Requisitos, restrições, critérios de aceitação "O app deve ter login com e-mail e senha"
PLAN.md Arquitetura, modelo de dados, stack tecnológica "Usar Flask + PostgreSQL + Redis"
TASK.md Tarefas atômicas com prioridade e estimativa "[ ] Criar endpoint POST /api/login (2h)"
Simulação: Editor SDD

Gerar com IA

Clique em "Gerar com IA" para usar o modelo Ollama e gerar ou refinar o conteúdo do documento. O Specx automaticamente injeta o contexto dos outros documentos SDD, regras, GEARS e memória.

Fluxo recomendado:
1. Escreva o SPEC.md primeiro (o que o projeto deve fazer)
2. Gere o PLAN.md com base no SPEC (como vai ser feito)
3. Gere o TASK.md com base no SPEC + PLAN (o que fazer agora)
4. Comece a trabalhar nas tarefas!

Perfis GEARS

GEARS é um framework de prompt engineering estruturado. Cada perfil define 5 dimensões que guiam como a IA deve se comportar:

Goals O que a IA deve realizar
Examples Exemplos concretos
Actions Passos a seguir
Rules Restrições rígidas
Style Tom e formatação
Perfis padrão
PerfilQuando usar
code_generatorGerar código limpo e aderente à SPEC.md
task_plannerQuebrar PLAN.md em tarefas atômicas
code_reviewerRevisar código contra os requisitos

Você pode carregar um perfil padrão ou criar o seu próprio na página GEARS.

Regras e Direcionamentos

Arquivos de regras são injetados no contexto da IA para garantir que ela siga as diretrizes do seu projeto. O Specx já cria um DEFAULT_RULES.md com regras básicas.

Exemplo de regras
# Regras do Projeto

## Estilo de Código
- Seguir PEP 8 para Python
- Usar type hints
- Funções com no máximo 30 linhas

## Arquitetura
- Separar responsabilidades
- Usar padrão MVC
- Nunca misturar lógica de negócio com apresentação

## Interação com IA
- Sempre referenciar SPEC.md antes de gerar código
- Gerar uma tarefa por vez de TASK.md
- Validar saída contra a arquitetura de PLAN.md
Dica: Você pode criar múltiplos arquivos de regras para diferentes contextos (ex: CODING_RULES.md, SECURITY_RULES.md). Todos são injetados automaticamente.

Bancos de Memória

Os bancos de memória permitem armazenar informações persistentes do projeto que a IA poderá consultar em todas as conversas.

Contexto
Estado atual, fatos chave, configurações
Decisões
Decisões arquiteturais com justificativa
Lições
O que funcionou e o que não funcionou
Conversas
Resumos de interações importantes
Como usar
  1. Acesse a aba de Memória no workspace
  2. Selecione o tipo de memória (contexto, decisões, lições, conversas)
  3. Adicione uma entrada com título, conteúdo e tags
  4. A IA terá acesso a essas informações em todas as próximas conversas
Importante: A memória é limitada a ~20 entradas mais recentes por tipo no contexto da IA. Use tags para organizar e busque entradas específicas quando necessário.

Gerenciar Modelos

O Specx se conecta ao Ollama local para listar e usar modelos. Você pode baixar novos modelos diretamente pela interface.

Modelos recomendados
ModeloTamanhoIdeal para
llama3~4.7 GBGeral, bom equilíbrio qualidade/velocidade
llama3:8b~4.7 GBVersão compacta do Llama 3
codellama~3.8 GBEspecializado em código
mistral~4.1 GBRápido e eficiente
phi3~2.2 GBLeve, para máquinas com menos RAM
Como baixar: Vá em Modelos → digite o nome → clique em Baixar. Ou use o terminal: ollama pull llama3

Perguntas Frequentes

Todos os dados ficam na pasta workspaces/ dentro do diretório do projeto. Cada workspace contém seus documentos SDD, regras, memórias e perfis GEARS.

Sim. Você pode desmarcar as opções de contexto no chat. A IA funcionará como um chatbot genérico sem contexto do projeto.

Ao clicar em "Gerar com IA" no editor SDD, o Specx monta um prompt completo com todos os contextos (outros documentos, regras, GEARS, memória) e envia para o modelo Ollama. O conteúdo gerado aparece diretamente no editor.

Atualmente não há exportação automática. Como os dados ficam em arquivos simples (.md e .json), você pode copiar a pasta do workspace manualmente para backup ou transferência.
